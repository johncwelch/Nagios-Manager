Host Manager

The Host Manager tab is where you manage hosts on the selected Nagios server. You can:

  • Add hosts

  • Get information about an existing host

Because of how complex Nagios dependencies can be (A host can be linked to host escalations, service escalations, services, etc.) and there being no effective way to resolve these within the Nagios API, for now, Nagios Manager cannot delete hosts. If the API is updated to allow for this, then the app will be too.


Host Manager Tab

As with the other tabs in Nagios Manager, the Host Manager contains the three basic  functional areas . The  main table  contains the (Nagios, not DNS) host name, IP Address/DNS Name, and the active status for every host on a given Nagios Server. (Note that when you first select the Host Manager tab, you see the warning about deleting hosts in the  status display .) When you select a server in the  main table , the following information is displayed in the  status display :

  • Display Name

  • Alias Name

  • Active Checks Enabled

  • Passive Checks Enabled

  • Notifications Enabled

  • Check Intervals (in minutes by default)

  • Retry Intervals (in minutes by default)

  • Max Check Attempts

  • Notification Interval (in minutes by default)

  • First Notification Delay (in minutes by default)

If you select a host and click the “Get Host Status” button, or double-click a host in the  main table , a HUD window appears with further information for that host:

  • Numerical Nagios Host ID for the selected host

  • Last Status

  • Last Status Update Time

  • Last Time Down

  • Last Time Unreachable

  • Last Check

  • Next Check

  • Flap Detection Enabled

  • Is Flapping?

  • Problem Acknowledged? (This is  only  for host problems, not service problems.)


Selecting a Server

The “Select Server” popup is where you select the specific server whose hosts you are managing. If there are multiple servers in  Nagios Manager’s preferences file , then the first server is decided by its order in the preferences file. The first host returned for a given server is selected by default in the  main table , while the  status display  will show the warning about being unable to delete hosts in Nagios Manager the first time you select the Host Manager tab. If you change servers, the  main table  will be reloaded with the hosts for the server you select.


Selecting and Getting more information for a Host

To select a host, click on that host’s row in the  main table . To get more information for that host beyond what is shown in the  status display , either double-click on that host in the  main table , or click the “Get Host Status” button.


Adding a Host

When adding a host, the Host Name and Address are required. For the Host Name, the DNS name is not required. For the Address, you can use either the DNS name or the IP Address of the host. Note there is  no  sanity checking of the address whatsoever. Any typos, the entering of an invalid IP address or DNS name will not be caught by Nagios Manager. That’s up to you. 

Once the host has been added, there can be a delay, up to six seconds, before Nagios Manager can pull that information. To refresh the list, re-select the Nagios Server you’re working against in the “Select Server:” popup list.

A note on contacts: the contacts list in the Host Manager tab allows for multiple selections so you can add multiple contacts when you create the host. 

The Hostgroup popup defaults to “None”, but allows you to put the host in one hostgroup when creating/adding a host. 

The Notification Period combo box allows you to select an already existing Notification Period, or enter one of your own. Note that entering one here will not set up the times/days for that notification period. You’ll have to go into that specific Nagios server and use the web UI to set up that information. 

By default, the Notification Options provided are for down, up, and recovery.
